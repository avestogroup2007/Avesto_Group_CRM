# Avesto Group CRM System — Стандарт дизайна

Единый ориентир для UI/UX. Основан на скилле **ui-ux-pro-max**
(`.claude/skills/ui-ux-pro-max`) и на уже сложившемся стиле приложения.
Цель — красиво, понятно и удобно для всего персонала: от рядового сотрудника
до администратора. Улучшения вносим **постепенно, не ломая этот стандарт**.

## Рекомендованный стиль

**Data-Dense Dashboard** — KPI-карточки, таблицы, сетка, максимум полезных
данных при спокойной подаче. Режимы: светлый (сейчас) и тёмный (в планах).
Уровень доступности — WCAG AA.

Шрифт — **Manrope**.

## Чек-лист перед изменением UI (skill pre-delivery)

- [ ] Иконки — SVG (Lucide), не эмодзи.
- [ ] `cursor: pointer` на всех кликабельных элементах.
- [ ] Ховер-состояния с плавным переходом 150–300 мс.
- [ ] Контраст текста ≥ 4.5:1 (мелкий вторичный текст — под контролем).
- [ ] Видимое кольцо фокуса для клавиатуры (`:focus-visible`).
- [ ] Уважать `prefers-reduced-motion`.
- [ ] Тач-таргеты ≥ 44×44px, зазор ≥ 8px.
- [ ] Адаптивность: 375 / 768 / 1024 / 1440.

## Что уже сделано

- **Слой доступности и взаимодействия** в `src/index.css`: кольцо фокуса для
  клавиатуры, корректные курсоры (в т. ч. фикс Tailwind v4), `prefers-reduced-motion`,
  утилита `.tnum` для табличных цифр. Внешний вид при работе мышью не изменился.

## Дорожная карта (по приоритетам скилла, внедряем по частям)

1. **Доступность (CRITICAL)** — ✅ слой фокуса/курсоров/reduced-motion.
   Далее: проверить контраст мелкого текста `faint (#94A3B8)`, добавить
   `aria-label` иконочным кнопкам, порядок табуляции.
2. **Тач и взаимодействие (CRITICAL)** — довести тач-таргеты до 44px на мобильном,
   единые ховер/нажатие/disabled состояния.
3. **Адаптивность (HIGH)** — аудит на 375px (канбан, таблицы касс), отсутствие
   горизонтального скролла, читаемость на телефоне.
4. **Типографика и числа (MEDIUM)** — применить `.tnum` к суммам, ценам и
   SLA-таймерам; выровнять шкалу размеров.
5. **Тёмная тема (MEDIUM)** — парная светлая/тёмная палитра для смен/ночной работы.
6. **Анимации (MEDIUM)** — единые длительности 150–300 мс, смысловые переходы фаз.

Каждый пункт — отдельный небольшой шаг с проверкой сборки и рендера, чтобы
не сломать стандарт.
